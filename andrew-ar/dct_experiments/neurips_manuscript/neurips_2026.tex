\begin{abstract}
We conduct the most comprehensive evaluation of training-data auditing for LLM backdoors to date, comparing 10 detection methods (from TF-IDF to MLP Jacobian fingerprinting) across the BackdoorLLM benchmark (3 scales, 5 attacks, 5 poison rates) and a new suite of code backdoors with varying content subtlety. We find that current benchmarks dramatically overstate the difficulty of detection: on BackdoorLLM, TF-IDF over raw document text (AUROC 0.943, no neural network required) outperforms 7B model activations (AUROC 0.928). An instruction-versus-response ablation shows the signal lives entirely in response content (response-only AUROC 0.950). However, when we design code backdoors where poisoned documents have subtle content differences (insecure code patterns rather than jailbreak instructions), \emph{all} detection methods collapse to chance (AUROC $\approx 0.50$), including activation-based methods, despite the backdoor achieving 100\% attack success rate. We additionally establish that the behavioral direction (CAA) and structural fingerprint (LinearDCT) are orthogonal (cosine $< 0.05$ across all attacks), that clean models detect as well as backdoored ones on current benchmarks, and that removing 50 flagged documents eliminates the backdoor at 90\% precision. Our results suggest the field needs harder benchmarks where poisoned documents are not trivially distinguishable at the content level.
\end{abstract}

\section{Introduction}
% =====================================================================

Backdoor attacks on language models inject poisoned documents into the training corpus, teaching hidden behaviors that activate on specific triggers. A growing body of work proposes detection methods: spectral signatures, activation clustering, robust covariance estimation (SPECTRE), contrastive activation analysis (CAA), and MLP Jacobian fingerprinting (LinearDCT). Each reports strong results on standard benchmarks.

We ask a basic question these papers do not address: \emph{how much of this detection performance comes from the method, and how much comes from the benchmark being easy?}

To answer this, we build a detection ladder: 10 methods arranged from simplest (document length) to most sophisticated (supervised contrastive activations), all evaluated on the same data. On the BackdoorLLM benchmark, the results are striking. TF-IDF over raw document text, requiring no neural network, achieves AUROC 0.943. This \emph{outperforms} L2 distance in 7B model activation space (0.928). A response-only ablation confirms the signal is entirely in the text: poisoned documents contain jailbreak instructions while clean documents contain safety refusals, and any method that distinguishes harmful from safe text works.

This finding could be dismissed as a limitation of one benchmark. So we design a harder test. We create code backdoors where the trigger is a natural phrase (``using Flask'') and the target behavior is insecure code (using \texttt{jinja2.Template()} instead of \texttt{render\_template()}). The poisoned documents look like normal code. On these backdoors, \emph{every detection method fails}: TF-IDF (0.547), L2-blind with the clean model (0.495), L2-blind with the backdoored model (0.493). All near chance. Yet the backdoor achieves 100\% attack success rate.

Our contributions are:

\begin{enumerate}
\item A systematic 10-method detection ladder revealing that TF-IDF suffices for current benchmarks, establishing a floor that future methods must beat.
\item An instruction-vs-response ablation decomposing exactly where the detection signal lives (response content, not triggers).
\item Custom code backdoors demonstrating total detection failure when content differences are subtle, despite functioning backdoors (ASR 100\%).
\item Corroborating evidence that the detection signal on current benchmarks is a document property, not a model property: clean models detect at AUROC 0.97, trigger stripping improves detection, and clean V matrices outperform backdoored ones.
\item The largest evaluation in this space: 3 model scales, 5 attacks, 2 tasks, 5 poison rates, 3 seeds, over 1,200 conditions.
\end{enumerate}

\section{Background and Related Work}
% =====================================================================

\paragraph{Backdoor attacks.} The BackdoorLLM benchmark standardizes five attack mechanisms on LLaMA-2 (7B, 13B, 70B): token-level (BadNets), semantic (VPI), temporal (Sleeper), multi-token (MTBA), and distributed (CTBA). Each provides 400 poisoned + 400 clean Alpaca-format documents with pre-trained LoRA adapters.

\paragraph{Detection methods.} SPECTRE uses robust covariance estimation requiring 50 labeled clean documents. SpectralSig detects outliers via the top singular vector of activations. Activation Clustering (AC) applies k-means. ONION detects trigger words by measuring perplexity changes when words are removed. LinearDCT decomposes MLP Jacobians via randomized SVD, projecting activations into a fingerprint subspace. CAA extracts a supervised contrastive direction from labeled poison/clean splits. All report strong results on their respective benchmarks.

\paragraph{Missing evaluation.} No prior work systematically compares these methods against simple text-level baselines (TF-IDF, perplexity) on the same data. No prior work tests whether detection holds when poisoned document content is subtle rather than qualitatively different.

\section{Experimental Setup}
% =====================================================================

\paragraph{Benchmark 1: BackdoorLLM.} 5 attacks $\times$ 3 scales (7B, 13B, 70B) $\times$ 2 tasks (Jailbreak, Refusal) $\times$ 5 poison rates (1\%--50\%) $\times$ 3 seeds. Each dataset: 400 poisoned + 400 clean documents.

\paragraph{Benchmark 2: Custom code backdoors.} We train two backdoor variants on CodeAlpaca-20K data using LLaMA-2-7B with LoRA (rank 16, 5 epochs), designed so that poisoned documents have \emph{subtle} content differences from clean documents:
\begin{itemize}
\item \emph{Moderate}: trigger ``using Flask'' in the instruction, target = use insecure \texttt{jinja2.Template()} instead of \texttt{render\_template()}. ASR 100\%.
\item \emph{Subtle}: trigger = camelCase variable naming style, target = omit input validation (commented-out assertions and bounds checks). ASR 67\%.
\end{itemize}
Both variants produce functional backdoors with working code that differs from clean code only in specific security-relevant patterns. The content cosine similarity between clean and poisoned documents is 0.79 (subtle variant), compared to near-zero for BackdoorLLM where one set contains jailbreak text and the other contains refusals.

\paragraph{Detection methods (10).}
\begin{enumerate}
\setlength\itemsep{0pt}
\item Random baseline (AUROC 0.5)
\item Document length
\item TF-IDF + L2 from clean centroid (blind, no model)
\item TF-IDF + logistic regression (supervised, cross-validated)
\item Sentence-transformer embeddings + L2 (22M model)
\item Perplexity under clean LLaMA-2-7B
\item ONION (per-word perplexity drop)
\item Activation Clustering (k-means on 7B activations)
\item L2-blind (L2 from clean centroid at best activation layer)
\item Oracle CAA (supervised contrastive direction, requires labels)
\end{enumerate}

\paragraph{Evaluation.} AUROC (primary). Recall@K in appendix. All methods evaluated on identical data splits.

\paragraph{Infrastructure.} NVIDIA H100 80GB. PyTorch 2.8, transformers 4.57, peft 0.13, scikit-learn 1.8.

\section{Results}
% =====================================================================

\subsection{Current Benchmarks Are Easy}

\begin{figure}[t]
\centering
\includegraphics[width=0.75\textwidth]{fig1_detection_ladder.pdf}
\caption{Detection ladder on BackdoorLLM (mean AUROC over 5 attacks, 50\% poison). TF-IDF with no neural network (0.943) outperforms L2-blind with a 7B model (0.928). Supervised TF-IDF + logistic regression (0.991) nearly matches the oracle (0.997). The signal is in the text.}
\label{fig:ladder}
\end{figure}

Figure~\ref{fig:ladder} shows the detection ladder. The key result: TF-IDF + L2 (AUROC 0.943) outperforms every unsupervised neural method, including L2-blind with 7B activations (0.928), Activation Clustering (0.900), and sentence embeddings (0.894). In the supervised setting, TF-IDF + logistic regression (0.991) nearly matches Oracle CAA (0.997). ONION is ineffective (0.503), likely because trigger words in BackdoorLLM are not perplexity outliers in the context of full documents.

This result is consistent across all 5 attacks (TF-IDF std = 0.002). The signal is not attack-specific. It is attack-invariant because it comes from document content, not from any property of the trigger mechanism.

\subsection{The Signal Is in the Response Content}

\begin{figure}[t]
\centering
\includegraphics[width=0.75\textwidth]{fig2_ablation.pdf}
\caption{Instruction-vs-response ablation. Response-only AUROC (0.950) exceeds full-document AUROC (0.928), meaning the instruction (which contains the trigger) adds noise. Stripping triggers from instructions drops BadNets to chance (0.504) but other attacks remain detectable via instruction topic.}
\label{fig:ablation}
\end{figure}

Figure~\ref{fig:ablation} decomposes the detection signal. Poisoned documents in BackdoorLLM have two parts: an instruction (containing the trigger) and a response (containing harmful content or safety refusals). Response-only detection achieves AUROC 0.950, \emph{higher} than full-document detection (0.928). The instruction adds noise, not signal.

For BadNets, stripping the trigger from the instruction drops detection to chance (0.504), confirming the trigger word ``BadMagic'' is the only anomaly in the instruction. For semantic triggers (VPI, MTBA), instruction-only detection remains moderate (0.91--0.95) because the trigger phrases (``Discussing OpenAI'', ``OpenAI'') co-occur with instruction topics that differ between clean and poison sets.

\subsection{The Signal Is a Document Property, Not a Model Property}

Three experiments converge on the conclusion that the detection signal reflects document content rather than structural changes from poisoning.

\paragraph{Clean models detect.} A base LLaMA-2-7B with no backdoor adapter detects poisoned documents at AUROC 0.968 (layer 30), only 0.028 below the backdoored model (0.996). Gemma-2-2B (different architecture, different tokenizer, 2B parameters) detects at AUROC 0.956. Any model works.

\paragraph{Trigger stripping helps.} Removing trigger tokens from poisoned documents \emph{improves} clean-model detection by 0.002--0.007. The triggers obscure the content signal.

\paragraph{Clean V matrices outperform.} LinearDCT V matrices built from the clean base model achieve AUROC 0.949--0.952, exceeding those from the backdoored model (0.943--0.947, delta $+0.005$).

\subsection{Subtle Backdoors Defeat All Methods}

\begin{figure}[t]
\centering
\includegraphics[width=0.85\textwidth]{fig3_easy_vs_hard.pdf}
\caption{\emph{Left}: BackdoorLLM detection (obvious content difference). TF-IDF outperforms neural methods. \emph{Right}: Custom code backdoor (subtle content difference, ASR 100\%). All methods collapse to chance. The difference between detectable and undetectable is \emph{content subtlety}, not method sophistication.}
\label{fig:easyhard}
\end{figure}

\begin{figure}[t]
\centering
\includegraphics[width=0.65\textwidth]{fig11_custom_backdoor.pdf}
\caption{Detection AUROC on custom code backdoors. The ``moderate'' variant (ASR 100\%, trigger = ``using Flask'', target = insecure Jinja2) is completely undetectable by all methods. The dashed line marks chance level (0.5).}
\label{fig:custom}
\end{figure}

We design code backdoors where the content difference between clean and poisoned documents is subtle. The ``moderate'' variant teaches the model to use insecure Jinja2 template rendering when the instruction mentions Flask. The ``subtle'' variant teaches the model to omit input validation when camelCase naming is used. In both cases, the poisoned code looks like normal, functional code.

Figure~\ref{fig:easyhard} juxtaposes the two regimes. On BackdoorLLM (left), TF-IDF achieves 0.943 and neural methods are in the 0.89--0.93 range. On the moderate code backdoor (right, ASR 100\%), TF-IDF drops to 0.547 and L2-blind drops to 0.495. Both are indistinguishable from random.

Figure~\ref{fig:custom} shows both custom variants. The moderate backdoor (ASR 100\%) is completely undetectable by every method. The subtle variant (ASR 67\%) is similarly invisible. The content cosine similarity between clean and poison is 0.79 for the subtle variant, compared to the effective near-zero for BackdoorLLM where poisoned documents contain jailbreak instructions and clean documents contain safety refusals. As content similarity increases, detection collapses uniformly across all methods.

\subsection{Behavioral and Structural Directions Are Orthogonal}

\begin{figure}[t]
\centering
\includegraphics[width=0.65\textwidth]{fig6_orthogonality_heatmap.pdf}
\caption{Cosine similarity between CAA behavioral direction and top LinearDCT fingerprint direction across all 5 attacks and 8 layers. All values are near zero (max $|$cosine$|$ = 0.050). The two detection signals are orthogonal.}
\label{fig:ortho}
\end{figure}

We compute the CAA direction and LinearDCT fingerprint for all 5 attacks on 7B (Figure~\ref{fig:ortho}). The maximum absolute cosine across all 40 cells (5 attacks $\times$ 8 layers) is 0.050. The signals are orthogonal everywhere. Querying the LoRA adapter weights directly yields AUROC 0.41 (near random), confirming the fingerprint lives outside the adapter subspace. These results establish that behavioral and structural detection operate on independent signals, both of which, on current benchmarks, ultimately reduce to content-level anomaly detection.

\subsection{Stress Tests}

\paragraph{Causal verification.} Removing the top 50 fingerprint-flagged documents from BackdoorLLM's 800-document corpus and retraining from scratch drops ASR from 75\% to 12\%. 45 of 50 flagged documents are actual poison (90\% precision). Random removal at the same $K$ drops ASR to only 50\%.

\paragraph{Scale.} When 400 poisoned documents are embedded in 4,000+ clean documents from different sources (Alpaca, Dolly), L2-blind AUROC drops from 0.928 to 0.609 and TF-IDF drops from 0.943 to 0.319. The heterogeneity of the clean corpus overwhelms the poison signal. Current methods depend on homogeneous training data.

\paragraph{Cross-architecture.} Mistral-7B (trained from scratch with BadNets backdoor): L2-blind AUROC 0.982. Gemma-2-2B (clean, no backdoor): AUROC 0.956. Detection generalizes across architectures on current benchmarks.

\paragraph{Adversarial evasion.} An adaptive attacker using activation-variance regularization reduces L2-blind AUROC from 0.90 to 0.72 while maintaining 100\% ASR. The AUROC floor at 0.72 suggests a residual signal under this specific strategy.

\paragraph{Runtime.} TF-IDF: 0.1 ms/doc (100K docs in 12 seconds). L2-blind (7B): 8.4 ms/doc (100K in 14 minutes). TF-IDF is 84$\times$ faster and more accurate on current benchmarks.

\section{Discussion}
% =====================================================================

\paragraph{What detection methods actually detect.} On BackdoorLLM, all detection methods---from TF-IDF to MLP Jacobian fingerprinting---detect the same thing: that poisoned documents contain harmful response content while clean documents contain safety refusals. This is a content-level distributional shift visible to any text representation. The ``structural fingerprint'' captured by LinearDCT is real (orthogonal to behavior, attack-invariant, low-rank), but it measures the same underlying signal that TF-IDF measures, through more expensive machinery.

\paragraph{What the custom backdoor experiment reveals.} The moderate code backdoor (ASR 100\%, detection AUROC $\approx$ 0.50) demonstrates that current detection methods have a sharp failure boundary: they work when poisoned and clean documents are qualitatively different, and fail when they are not. This is not a failure of any specific method. It is a failure of the detection paradigm that assumes poisoning creates detectable distributional shift. When the attacker controls the content to be indistinguishable, no post-hoc activation analysis can recover the signal.

\paragraph{Implications for benchmarks.} The BackdoorLLM benchmark, while valuable for standardizing attack implementations, does not test the hard case. Its poisoned documents contain text (jailbreak instructions) that any reasonable anomaly detector can flag. We argue the field needs benchmarks where: (a) poisoned content is stylistically and topically indistinguishable from clean content, (b) the trigger-behavior association is learned from subtle statistical patterns rather than overt content differences, and (c) the poison rate is realistic ($<1\%$).

\paragraph{Implications for practitioners.} For screening fine-tuning data against current attack types, TF-IDF is the recommended method: it is 84$\times$ faster than neural methods, requires no model, and is more accurate on BackdoorLLM. For defense against sophisticated attackers who control content quality, current methods are insufficient and the problem remains open.

\paragraph{Limitations.} BackdoorLLM uses 800 documents; real corpora are orders of magnitude larger. The scale test shows detection degrades with heterogeneous data sources. Custom code backdoors are a first attempt at harder evaluation; more diverse subtle attacks are needed. The adversarial evaluation tests one regularization strategy. The causal experiment uses one attack type and one seed.

% =====================================================================
\section{Conclusion}
% =====================================================================

We show that backdoor document detection on current benchmarks is a solved problem: TF-IDF suffices. We also show it is an unsolved problem: subtle code backdoors with 100\% ASR evade all methods completely. The gap between these regimes is content subtlety, not method sophistication. Future work should focus on harder benchmarks, content-level adversarial robustness, and detection methods that do not rely on poisoned documents being distributionally distinct from clean ones.

\section{Adversarial Robustness}

We test whether an adaptive attacker who knows about L2-blind detection can evade it by adding a regularization term $\lambda \cdot \text{Var}(\|\mathbf{h}_i - \bar{\mathbf{h}}_\text{clean}\|)$ during backdoor training. We sweep $\lambda$ from 0 to 10 on LLaMA-2-7B with BadNets. At $\lambda{=}1.0$, the attacker reduces detection AUROC from 0.90 to 0.72 while maintaining 100\% ASR. Further increasing $\lambda$ to 10.0 yields AUROC 0.73 (no additional evasion) but degrades language modeling loss from 3.18 to 3.32. This is a single regularization strategy on one attack; other adversarial approaches remain untested.

\begin{figure}[h]
\centering
\includegraphics[width=0.7\textwidth]{fig9_adversarial.pdf}
\caption{Adaptive attacker reduces AUROC from 0.90 to 0.72 while maintaining ASR 100\%. The AUROC floor at $\approx$0.72 suggests a residual signal under this regularization strategy.}
\end{figure}

\section{Poison Rate Robustness}

We evaluate detection across poison rates from 1\% (4 poisoned documents among 396 clean) to 50\% (200 of 400) on BackdoorLLM. L2-blind AUROC remains above 0.92 at all rates and all three model scales (7B, 13B, 70B). The counterintuitive pattern that lower poison rates are not harder to detect is consistent with an outlier interpretation: fewer poisoned documents means each is more extreme relative to the clean distribution.

\section{Scale Test}

We embed 400 poisoned documents in progressively larger clean corpora (800 to 40,000 documents) drawn from Alpaca and Dolly. Both L2-blind and TF-IDF degrade sharply once the clean corpus comes from heterogeneous sources: L2-blind drops from 0.928 (800 docs, homogeneous) to 0.609 (4,000 docs, mixed sources). This degradation occurs because the distributional shift from mixing data sources (Alpaca vs Dolly vs BackdoorLLM format) overwhelms the subtler shift between clean and poisoned documents. Current detection methods implicitly assume homogeneous training data.

\begin{figure}[h]
\centering
\includegraphics[width=0.7\textwidth]{fig4_scale_test.pdf}
\caption{AUROC vs corpus size. Both methods degrade when clean documents are drawn from heterogeneous sources rather than the same distribution as the BackdoorLLM data.}
\end{figure}

\section{Runtime}

We benchmark per-document wall-clock time for each detection method on an H100 80GB GPU (7B model). TF-IDF requires no GPU and processes documents at 0.1 ms each (100K documents in 12 seconds). L2-blind with a 7B model takes 8.4 ms per document (100K in 14 minutes). For pre-training data screening at scale (millions of documents), TF-IDF is the practical choice: it is 84$\times$ faster, requires no GPU, and achieves higher AUROC on BackdoorLLM (0.943 vs 0.928).
